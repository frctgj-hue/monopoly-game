# 🔄 Исправление переподключения к серверу

## Дата: 2026-05-09

## Проблема
При потере соединения с сервером появлялся экран "Подключение к серверу", после чего кнопки игры переставали работать и игра не могла быть продолжена.

## Причины
1. При переподключении происходила перезагрузка страницы (`window.location.reload()`)
2. Состояние игры терялось
3. Не было механизма восстановления состояния после переподключения

## Исправления

### 1. Убрана перезагрузка страницы (useSocket.ts)
```typescript
// БЫЛО:
socket.on('reconnect', (attemptNumber) => {
  if (attemptNumber > 1) {
    window.location.reload(); // ❌ Теряет состояние
  }
});

// СТАЛО:
socket.on('reconnect', (attemptNumber) => {
  setConnected(true);
  // ✅ НЕ перезагружаем - сохраняем состояние
});
```

### 2. Добавлен механизм восстановления состояния (App.tsx)
```typescript
socket.on('connect', () => {
  // Если у нас есть gameState, восстанавливаем игру
  if (gameState && gameState.id) {
    socket.emit('rejoin-game', { gameId: gameState.id }, (response) => {
      if (response.success) {
        setGameState(response.game); // ✅ Восстанавливаем состояние
        setPhase('playing');
      }
    });
  }
});
```

### 3. Добавлен обработчик rejoin-game на сервере (index.ts)
```typescript
socket.on('rejoin-game', ({ gameId }, callback) => {
  const game = gameService.getGame(gameId);
  if (!game) {
    callback({ success: false, message: 'Игра не найдена' });
    return;
  }

  const player = game.players.find(p => p.id === socket.id);
  if (!player) {
    callback({ success: false, message: 'Игрок не найден' });
    return;
  }

  socket.join(gameId); // ✅ Переподключаем к комнате
  callback({ success: true, game }); // ✅ Возвращаем состояние
});
```

### 4. Улучшен экран подключения (App.tsx)
```typescript
// Показывает разные сообщения:
// - "Подключение к серверу..." - первое подключение
// - "Переподключение..." - восстановление соединения
// - "Восстанавливаем состояние игры..." - если есть gameState
```

## Как это работает

### Сценарий 1: Потеря соединения во время игры

1. **Соединение потеряно** → Показывается экран "Переподключение..."
2. **Socket.io автоматически переподключается** (настроено в useSocket)
3. **Срабатывает событие 'connect'** → Проверяется наличие gameState
4. **Отправляется запрос rejoin-game** → Сервер возвращает актуальное состояние
5. **Состояние восстановлено** → Игра продолжается с того же места

### Сценарий 2: Первое подключение

1. **Соединение устанавливается** → Показывается "Подключение к серверу..."
2. **gameState === null** → Переподключение не требуется
3. **Показывается лобби** → Пользователь создает/присоединяется к игре

## Настройки переподключения

В `useSocket.ts` настроены параметры Socket.io:

```typescript
{
  reconnection: true,              // ✅ Автоматическое переподключение
  reconnectionDelay: 500,          // Задержка 0.5 сек
  reconnectionDelayMax: 2000,      // Максимум 2 сек
  reconnectionAttempts: Infinity,  // Бесконечные попытки
  timeout: 20000,                  // Таймаут 20 сек
  transports: ['websocket', 'polling'], // WebSocket + fallback
  pingTimeout: 60000,              // Пинг таймаут 60 сек
  pingInterval: 25000              // Пинг каждые 25 сек
}
```

## Тестирование

### Как протестировать переподключение:

1. **Запустите игру** и начните играть
2. **Симулируйте потерю соединения**:
   - Откройте DevTools (F12)
   - Network → Offline (включите офлайн режим)
   - Подождите 5 секунд
   - Network → Online (выключите офлайн режим)

3. **Что должно произойти**:
   - ✅ Появляется экран "Переподключение..."
   - ✅ Через 1-2 секунды соединение восстанавливается
   - ✅ Игра продолжается с того же места
   - ✅ Все кнопки работают
   - ✅ Состояние игры сохранено

## Возможные проблемы

### Проблема 1: "Игра не найдена" после переподключения
**Причина**: Сервер перезапустился и потерял состояние игры

**Решение**: 
- Используйте персистентное хранилище (Redis, MongoDB)
- Или создайте новую игру

### Проблема 2: Кнопки все еще не работают
**Причина**: Состояние не восстановилось полностью

**Решение**:
1. Проверьте логи в консоли
2. Убедитесь, что `rejoin-game` возвращает `success: true`
3. Проверьте, что `setGameState(response.game)` вызывается

### Проблема 3: Постоянные переподключения
**Причина**: Проблемы с сетью или сервером

**Решение**:
1. Проверьте стабильность интернет-соединения
2. Проверьте, что backend работает
3. Проверьте логи сервера
