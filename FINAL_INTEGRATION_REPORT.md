## 🎮 Как работают анимации

### 1. Движение фишки
**Триггер**: Событие `dice-rolled` от сервера  
**Процесс**:
1. Board.tsx отслеживает изменение позиции игрока
2. Создается TokenPathAnimation с fromPosition и toPosition
3. Фишка плавно перемещается через каждую клетку
4. После завершения фишка появляется на конечной позиции

### 2. Переворачивание карточки
**Триггер**: Событие `card-drawn` от сервера  
**Процесс**:
1. CardModal появляется с рубашкой карточки
2. Через 300ms начинается 3D переворот (rotateY 180deg)
3. Показывается лицевая сторона с текстом карточки
4. Пользователь нажимает "Продолжить"

### 3. Победа
**Триггер**: Событие `game-finished` от сервера  
**Процесс**:
1. App.tsx сохраняет победителя в состояние
2. VictoryAnimation появляется поверх всего
3. Генерируется 50 частиц конфетти
4. Показывается корона, фишка, статистика, фейерверк
5. При закрытии возврат в лобби

### 4. Банкротство
**Триггер**: Событие `player-bankrupt` от сервера  
**Процесс**:
1. App.tsx сохраняет банкрота в состояние
2. BankruptcyAnimation появляется с тряской
3. Фишка игрока затухает и исчезает
4. Через 2 секунды автоматически закрывается

## 📝 Технические заметки

### Архитектурные решения

1. **Абсолютное позиционирование для TokenPathAnimation**
   - Позволяет фишке двигаться поверх игрового поля
   - Использует координаты DOM элементов для точного позиционирования
   - z-index: 50 для отображения поверх всех элементов

2. **CSS 3D трансформации для карточек**
   - perspective: 1000px для глубины
   - transform-style: preserve-3d для сохранения 3D пространства
   - backface-visibility: hidden для скрытия задней стороны

3. **Генерация конфетти через JavaScript**
   - 50 частиц с случайными параметрами
   - Разные цвета, задержки, длительности
   - CSS анимация для падения

4. **Автоматическое управление жизненным циклом**
   - BankruptcyAnimation автоматически закрывается через 2s
   - VictoryAnimation требует действия пользователя
   - Очистка таймеров при размонтировании
